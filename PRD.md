# 个人作品集网站 — 产品需求文档（PRD v1.0）

> 面向设计师 / 开发者的专业形象展示平台  
> 最后更新：2026-04-23

---

## 01 产品概述

### 产品定位

个人作品集网站是设计师与开发者在互联网上的「数字名片」，也是获取客户、工作机会的核心转化渠道。本产品定位为极简高质感的静态展示网站，以视觉冲击力和内容说服力为核心，体现专业技能与审美水准。

**核心价值主张：** 在 5 秒内让访客理解「你是谁、你做什么、你有多厉害」，并驱动其发起联系。

### 目标用户画像

| 用户类型 | 场景描述 | 核心诉求 | 行为特征 |
|---------|---------|---------|---------|
| 潜在客户 / 甲方 | 企业主或产品负责人，正在寻找外包设计或开发 | 快速判断专业程度与风格匹配度 | 只浏览 Hero + 精选作品，平均停留 < 2 分钟 |
| 招聘 HR / 猎头 | 通过 LinkedIn 或社交渠道访问 | 验证技能真实性，获取联系方式 | 重点查看「关于我」和技能标签 |
| 同行 / 创作者 | 设计社区、Twitter 分享引流 | 学习参考、建立连接 | 深度浏览，会分享或收藏 |
| 网站本人 | 持续维护更新内容 | 低成本更新作品、快速发布 | 需要清晰的内容管理方式 |

## 04 界面设计要求

### 视觉风格

- **风格定位：** 极简主义（Minimalism）+ 高端感
- **主色：** 深黑 `#09090B`，背景白 `#FAFAFA`
- **强调色：** 品牌蓝 `#2563EB`，用于 CTA 与高亮元素
- **中性色：** Zinc 灰阶系列（zinc-100 / zinc-400 / zinc-500 / zinc-900）
- **禁止使用：** 彩虹色、文字渐变、霓虹效果

### 字体规范

- **标题字体：** Archivo（现代感、冲击力）
- **正文字体：** Space Grotesk（易读、技术感）
- **标题字号：** H1 ≥ 64px（桌端），响应式缩放
- **正文字号：** 16–18px，行高 1.6–1.8
- **字重：** 300 / 400 / 500 / 600 / 700，避免 800+

### 间距与布局

- **最大容器宽度：** `max-w-7xl`（1280px）
- **水平内边距：** `px-4`（移动端）→ `px-8`（桌端）
- **Section 间距：** `py-24`（96px）
- **卡片圆角：** `rounded-2xl` / `rounded-3xl`
- **导航栏：** 悬浮卡片式，四边留 16px 间距

### 动效规范

- **Hover 过渡：** 150–300ms ease，禁止超过 500ms
- **卡片悬停：** `translateY(-8px)` + cubic-bezier 弹性曲线
- **入场动画：** fade-in + slide-up，延迟分组加载
- **无障碍：** 必须支持 `prefers-reduced-motion` 关闭所有动效
- **禁止：** 闪烁、旋转过载、全屏遮罩动画

### 响应式断点

| 断点 | 宽度范围 | 布局变化 | 关键调整 |
|-----|---------|---------|---------|
| Mobile | 375–767px | 单列布局，汉堡菜单 | 字号缩小，卡片全宽，CTA 纵向排列 |
| Tablet | 768–1023px | 两列作品网格 | 导航展开，About 双栏保留 |
| Desktop | 1024–1279px | 三列作品网格 | 全功能展示，导航完整 |
| Wide | 1280px+ | 内容居中 max-w-7xl | 增加 Hero 装饰元素，字号更大 |

## 05 技术栈建议

### 方案 A — 纯静态（推荐新手，当天可上线）

| 层次 | 技术 | 理由 |
|-----|-----|-----|
| 结构 | HTML5 | 零依赖，极速加载 |
| 样式 | Tailwind CSS CDN | 无需构建步骤，快速开发 |
| 字体 | Google Fonts | 免费高质量字体 |
| 图片 | Unsplash / 本地 | 占位图快速替换 |
| 部署 | Vercel / Netlify | 免费，一键部署，自动 HTTPS |
| 表单 | Formspree | 无后端联系表单 |

### 方案 B — Next.js（推荐有经验者，长期维护）

| 层次 | 技术 | 理由 |
|-----|-----|-----|
| 框架 | Next.js 14（App Router） | SSG 生成，SEO 友好 |
| 样式 | Tailwind CSS | 工程化，可维护 |
| 内容管理 | MDX / Contentlayer | Markdown 驱动作品内容 |
| 动效 | Framer Motion | 声明式动画，简单易用 |
| 图片 | next/image | 自动优化，WebP 转换 |
| 部署 | Vercel | 与 Next.js 原生集成 |

> **选型建议：** 如果目标是「快速上线展示」，选方案 A，当天可发布。如果希望长期维护、SEO 优化、博客扩展，选方案 B。现有的 `portfolio.html` 属于方案 A，可直接部署使用。

### 可选增强工具

| 工具 | 用途 | 推荐指数 |
|-----|-----|---------|
| GSAP / Framer Motion | 高级滚动动画、视差效果 | 推荐（有需要时） |
| Lottie | JSON 矢量动画，如 loading、插图 | 可选 |
| Google Analytics 4 | 访客数据统计，了解用户行为 | 强烈推荐 |
| Hotjar | 热力图分析，优化页面布局 | 可选 |
| Cloudflare Pages | 全球 CDN，国内访问速度更快 | 国内用户推荐 |

## 06 非功能性需求

### 性能指标（Core Web Vitals）

| 指标 | 目标值 | 评级 | 优化手段 |
|-----|-------|-----|---------|
| LCP（最大内容绘制） | < 2.5 秒 | Good | Hero 图片预加载，使用 WebP 格式 |
| INP（交互延迟） | < 100ms | Good | 减少主线程阻塞，懒加载非关键 JS |
| CLS（布局偏移） | < 0.1 | Good | 图片指定宽高，字体 font-display: swap |
| TTFB（首字节时间） | < 600ms | 推荐 | 静态部署 + CDN 边缘节点 |
| Lighthouse 综合评分 | ≥ 90 分 | 优秀 | 以上优化综合保障 |

### 可访问性（Accessibility）

| 要求 | 标准 | 实现方式 |
|-----|-----|---------|
| 文字对比度 | WCAG AA（4.5:1） | 正文用 `#09090B` on `#FAFAFA`，确保最低比例 |
| 键盘导航 | 所有交互可键盘操作 | 正确的 tabindex，focus ring 可见 |
| 屏幕阅读器 | 语义化 HTML 结构 | 使用 nav/main/section/article 等标签 |
| 图片 Alt 文本 | 所有图片有描述性文字 | 作品图提供项目名称和类型描述 |
| 减少动效支持 | prefers-reduced-motion | CSS media query 禁用所有过渡动画 |

### SEO 要求

| 项目 | 要求 | 优先级 |
|-----|-----|-------|
| 页面标题 | 唯一、包含关键词（姓名 + 职业） | P0 |
| Meta Description | 150 字以内，含核心技能关键词 | P0 |
| OG 标签 | 社交分享时展示正确封面图和描述 | P1 |
| 语义化 URL | 作品详情页用 `/work/project-name` 格式 | P1 |
| Sitemap.xml | 自动生成，提交 Google Search Console | P2 |
| Structured Data | Person schema，增强搜索展示 | P2 |

### 安全与隐私

| 类别 | 要求 |
|-----|-----|
| HTTPS | 强制 HTTPS，HTTP 301 重定向 |
| 外链安全 | 第三方链接添加 `rel="noopener noreferrer"` |
| 邮件防爬 | 不明文写邮箱地址，使用编码或表单替代 |
| 图片版权 | 自拍照片 + 授权图库 + CC0 图片，不使用未授权图 |
| 隐私合规 | 如启用 Analytics，需添加 Cookie 告知提示（GDPR） |
